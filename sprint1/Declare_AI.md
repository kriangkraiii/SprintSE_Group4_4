# การประกาศการใช้ปัญญาประดิษฐ์ (AI Declaration)

รายงานฉบับนี้ เป็นส่วนหนึ่งของการดำเนินโครงการในรายวิชา โดยอ้างอิงประกาศมหาวิทยาลัยขอนแก่น เรื่อง หลักการและแนวทางการใช้ปัญญาประดิษฐ์ ข้าพเจ้าขอรับรองว่าการนำ AI มาใช้ในงานนี้ เป็นไปเพื่อวัตถุประสงค์ในการสนับสนุนการเรียนรู้ เพิ่มประสิทธิภาพการทำงานในฐานะเครื่องมือช่วยคิดวิเคราะห์ (Co-Pilot) และตรวจสอบความถูกต้อง (Reviewer) โดยข้าพเจ้าได้ทำการตรวจสอบความถูกต้องของผลลัพธ์ทั้งหมดด้วยตนเอง และขอประกาศรายละเอียดการใช้งานดังนี้

---

## 1. วัตถุประสงค์และการใช้งาน
ข้าพเจ้าใช้งานปัญญาประดิษฐ์ในฐานะ **"ผู้ช่วยวิจัยและพัฒนา" (Research & Development Assistant)** โดยใช้สนับสนุนในขั้นตอนต่างๆ ดังนี้:

### (1) การวางแผนและออกแบบ (Planning & Architecture)
- **เครื่องมือ**: Agentic IDE 4.6 (Model: Opus) และ Brainstorm Module
- **การใช้งาน**: ใช้ปัญญาประดิษฐ์ในการช่วย Brainstorm โครงสร้างโปรเจค (Project Structure) แบบ Monorepo เพื่อแยกส่วน Frontend และ Backend ให้เป็นระเบียบตามหลัก Software Engineering
- **การตรวจสอบ**: ข้าพเจ้าเป็นผู้ตัดสินใจเลือกโครงสร้างสุดท้าย และปรับปรุง Directory Structure ให้เหมาะสมกับ Library ที่ใช้จริง (Nuxt 3 + Express)

### (2) การเขียนโค้ดและแก้ไขข้อผิดพลาด (Development & Debugging)
- **เครื่องมือ**: Agentic IDE (Models: Opus, 5.2)
- **การใช้งาน**: 
  - ใช้ช่วยร่างโครงสร้างของ API Endpoints และ Database Schema (Prisma Schema) เบื้องต้น
  - ใช้ช่วยวิเคราะห์ Error Log และแนะนำแนวทางการแก้ไข (Root Cause Analysis) เมื่อเกิดบั๊กซับซ้อน เช่น ปัญหาการตั้งค่า Environment Variables หรือ CORS Policy
  - ใช้ช่วยเขียน Script พื้นฐาน (เช่น `npm run generate`, `npm run build`)
- **การตรวจสอบ**: ข้าพเจ้าได้นำโค้ดที่ AI แนะนำมาเขียนทดสอบการทำงานจริง (Manual Testing) ปรับ Logic ให้สอดคล้องกับ Business Requirement และแก้ไขบั๊กที่ AI แนะนำไม่ถูกต้องด้วยตนเอง

### (3) การจัดทำเอกสารและรายงาน (Documentation & Reporting)
- **เครื่องมือ**: Agentic IDE (Modules: Write Change Log, Documentation Helper)
- **การใช้งาน**: ใช้ช่วยเรียบเรียงประวัติการแก้ไข (Change Log) จาก Git Commit History และช่วยร่างโครงร่างของไฟล์ README.md ให้มีความเป็นมืออาชีพ
- **การตรวจสอบ**: ข้าพเจ้าได้ตรวจทานเนื้อหาภาษาไทยและภาษาอังกฤษ แก้ไขรายละเอียดทางเทคนิค และเพิ่มเติมข้อมูลเฉพาะ (Specific Implementation Details) ที่ AI ไม่สามารถระบุได้

### (4) การทดสอบและการติดตั้ง (Testing & Deployment)
- **เครื่องมือ**: Agentic IDE (Modules: API Command Basic Test, Model 5.2 for Deployment)
- **การใช้งาน**: ใช้ช่วยแนะนำการตั้งค่า Server Environment, การเขียน Unit Test เบื้องต้น (Test Cases), และคำสั่งพื้นฐานในการ Deploy Web Application
- **การตรวจสอบ**: ข้าพเจ้าได้ทำการรัน Test Script จริง และปรับแต่งการตั้งค่า Server ให้เหมาะสมกับ Hosting ที่ใช้งาน

## 2. การตรวจสอบความถูกต้อง (Verification Statement)
ข้าพเจ้าขอยืนยันว่า **"ผลงานที่ส่งนี้ไม่ได้เกิดจากการให้ AI สร้างให้ทั้งหมด (Generative Creation) โดยปราศจากการตรวจสอบ"** แต่เป็นการใช้ AI เพื่อ:
1.  **ปรับปรุงคุณภาพโค้ด (Refactoring)** ให้สะอาดและอ่านง่ายขึ้น
2.  **ตรวจสอบความผิดพลาด (Debugging)** เพื่อลดเวลาในการแก้ปัญหา
3.  **สนับสนุนข้อมูล (Knowledge Support)** เพื่อทำความเข้าใจเทคนิคใหม่ๆ ในการเขียนโปรแกรม

ผลลัพธ์สุดท้ายที่ปรากฏในชิ้นงาน ได้ผ่านการตรวจสอบความถูกต้อง และปรับปรุงแก้ไขเนื้อหาส่วนที่ AI อาจให้ข้อมูลคลาดเคลื่อนโดยข้าพเจ้าเรียบร้อยแล้ว
